# 物联网接入标准

## 文档状态

- 状态：Draft v0
- 目标：冻结硬件接入、MQTT topic、状态回传和 OTA 的最小标准
- 主线：为设备控制闭环服务

## 接入原则

- 所有设备统一以 `device:{device_id}` 作为逻辑身份
- 所有 MQTT payload 必须满足 RakMessage 语义
- MQTT 只负责命令与状态通道，不承载复杂业务编排
- 设备端必须做到可订阅、可执行、可回传、可心跳

## 设备标识

| 项目 | 规则 |
| --- | --- |
| device_id | 全局唯一，例如 `esp32-001` |
| source | 设备发消息时固定为 `device:{device_id}` |
| target | 设备收命令时固定为 `device:{device_id}` |

## Topic 规范

```text
rak/{device_id}/cmd
rak/{device_id}/state
rak/{device_id}/event
rak/{device_id}/heartbeat
rak/{device_id}/ota
```

## Topic 语义

| Topic | 方向 | 用途 |
| --- | --- | --- |
| `rak/{device_id}/cmd` | 云层 → 设备 | 下发动作命令 |
| `rak/{device_id}/state` | 设备 → 云层 | 上报执行状态 |
| `rak/{device_id}/event` | 设备 → 云层 | 上报设备事件 |
| `rak/{device_id}/heartbeat` | 设备 → 云层 | 上报在线心跳 |
| `rak/{device_id}/ota` | 云层 → 设备 | 下发 OTA 指令 |

## 命令消息

示例：

```json
{
  "version": "v0",
  "type": "action",
  "trace_id": "9f0e7b14-bf58-4b6a-9a2a-8f4e7f8e6001",
  "source": "cloud:go-gateway",
  "target": "device:esp32-001",
  "action": "lock_open",
  "params": {
    "timeout_ms": 3000
  },
  "data": {},
  "timestamp": 1710000002
}
```

设备侧要求：

- 收到命令后先校验 `version`
- 校验 `action` 是否在本设备支持范围内
- 保留原始 `trace_id`
- 执行完成后向 `state` topic 回传结果

## 状态消息

示例：

```json
{
  "version": "v0",
  "type": "state",
  "trace_id": "9f0e7b14-bf58-4b6a-9a2a-8f4e7f8e6001",
  "source": "device:esp32-001",
  "target": "cloud:go-gateway",
  "action": "",
  "params": {},
  "data": {
    "device_id": "esp32-001",
    "status": "done",
    "result": "ok"
  },
  "timestamp": 1710000004
}
```

`data.status` 建议取值：

- `queued`
- `running`
- `done`
- `failed`

## 事件消息

示例：

```json
{
  "version": "v0",
  "type": "event",
  "trace_id": "event-doorbell-1710000010",
  "source": "device:esp32-001",
  "target": "cloud:go-gateway",
  "action": "",
  "params": {},
  "data": {
    "event_name": "doorbell_pressed"
  },
  "timestamp": 1710000010
}
```

## 心跳消息

示例：

```json
{
  "version": "v0",
  "type": "event",
  "trace_id": "heartbeat-esp32-001-1710000015",
  "source": "device:esp32-001",
  "target": "cloud:go-gateway",
  "action": "",
  "params": {},
  "data": {
    "event_name": "heartbeat",
    "online": true
  },
  "timestamp": 1710000015
}
```

## OTA 消息

OTA topic：

```text
rak/esp32-001/ota
```

Payload：

```json
{
  "version": "v0",
  "type": "action",
  "trace_id": "ota-esp32-001-1710000020",
  "source": "cloud:go-gateway",
  "target": "device:esp32-001",
  "action": "device_ota",
  "params": {
    "firmware_version": "v0.1.0",
    "artifact_url": "https://example.com/firmware/v0.1.0.bin"
  },
  "data": {},
  "timestamp": 1710000020
}
```

OTA 回执：

- 下载开始时上报 `state=running`
- 下载成功时上报 `state=done`
- 下载失败时上报 `state=failed`

## 设备最小执行引擎

设备侧至少实现以下逻辑：

1. 连接 MQTT Broker
2. 订阅 `cmd` 和 `ota`
3. 解析 JSON
4. 执行动作
5. 向 `state` 或 `event` 回传结果
6. 周期性上报 `heartbeat`

## 安全边界

- 正式环境默认使用 TLS
- `example/物联网标准.ipynb` 中关闭证书校验的方式只允许作为临时调试，不进入正式标准
- 设备凭据与证书分发方案不在本轮 MVP 文档展开，但必须为后续保留独立配置能力

## 联调验收

- Go 能成功向 `cmd` 下发 RakMessage
- 设备能按原样保留 `trace_id` 回传状态
- 前端能看到与设备 `trace_id` 对应的执行状态
- 同一设备至少完成一次动作执行和一次状态回传
